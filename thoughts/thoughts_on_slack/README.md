# Thoughts on using slack effectively

## First of all

Whenever somebody writes anything in a channel, consider this single post as start of a discussion about a topic. Therefore any reply should be done in a thread.

If every reply is done in the main thread, it will be filled up with meaningless comments and the “topics” will be lost in the sea of “yes”, “ok”, “me to” and other unrelated chats. Worst case is when multiple discussions overlap, and the reader has to "re-thread" the conversations on the fly.

This also means that whenever you write something directly in a channel, consider the wording as a topic. This means consider how to phrase it short and precise.

## Secondly

If all you want is to agree or disagree to something, do not write “OK” or “Disagree”. Use thumbs up and down icon. It gets the job done, and takes up minimal space. Also it is possible for everybody to see who agreed and disagreed.

## Thirdly

And this might be controversial for some. Whenever you consider doing a private chat with one or two, just don’t!

If this conversation turns out to be interesting for everybody it is a hassle to move it later on.

And in a collaborative work environment, there shouldn’t be any “secret” discussions.

Instead do a post in an appropriate channel (with a descriptive topic of course) and @ the persons you want to participate.

This has the benefit that newcomers on any project can read up on what has been discussed so far.

## Oh and of course.

If you cannot find the appropriate channel, it might be missing. So create it.

This is a nice feature in Slack. Anybody and everybody can create channels.

This of course also means that from time to time channels should be archived if they are not used any more.

## Best of all

When you make a mistake, correct it. All messages can be edited and deleted.

So if you forgot to reply in a thread, delete your message and reply in the thread.

Feel a discussion got to far of topic, change the main message to reflect that, so anybody reading this later can make sense of it.

## Are you nuts? Some things ARE secret!

Well of course some things needs to be kept confidential, like sensitive information about customers.

But if that is the case, create a channel for that subject with restrictions on who can participate. Then later these restrictions can be changed or lifted. Or newcomers can be added, and catch up on what has been going on.

Also real "private" discussions are ok, and should be done as DM's. But in the sense that they are private not secret or confidential. (Private = nothing to do with work.)
