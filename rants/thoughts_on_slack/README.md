# Thoughts on using slack effectively

## First of all:

Whenever somebody writes anything in a channel, consider this single post as start of a discussion about a topic. Therefor any reply should be done in a thread.

If every reply is done in the main thread, it will be filled up with meaningless comments and the “topics” will be lost in the sea of “yes”, “ok”, “me to” and so on.

This also means that whenever you write something directly in a channel, consider the wording as a topic. This means consider how to fraise it short and precise.

## Secondly

If all you want is to agree or disagree to something, do not write “OK” or “Disagree”. Use thumbs up and down icon. It gets the job done, and takes up minimal space. Also it is possible to for everybody to see who agreed and disagreed.

## Thirdly

And this might be controversial for some. Whenever you consider doing a private chat with one or two, just don’t!

If this conversation turns out to be interesting for everybody it is a hassle to move it later on.

And in a collaborative work environment, there shouldn’t be any “secret” discussions.

Instead do a post in an appropriate channel (with a descriptive topic of course) and @ the persons you want to participate.

This has the benefit that newcomers on any project can read up on what has been discussed so far.

## Oh and of course.

If you cannot find the appropriate channel, it might be missing. So create it.

This is where Slack shines. Anybody and everybody can create channels.

This of course also means that from time to time channels should be archived if they are not used anymore.

## Best of all

When you make a mistake, correct it. All messages can be edited and deleted.

So if you forgot to reply in a thread, delete your message and reply in the thread.

Feel a discussion got to far of topic, change the main message to reflect that, so anybody reading this later can make sense of it.

I promise if we do all this (and more), slack will become a valuable store of common knowledge, and without much effort!

## Q and A

Q: What if I need to have an confidential discussion with a couple of other people, isn't it then okay to have a private channel?

A: Yes, but create a topic Chanel with restrictions on who can participate. Then later these restrictions can be changed or lifted.
   But real private discussions are ok. In that they are private not secret. Private=nothing to do with work.
